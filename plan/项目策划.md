# 基于flutter/dart的短险卡记录软件

## 简介
这是一个基于flutter/dart的短险卡记录软件，具有以下特点：
1. UI界面使用Material Design 3，支持Android平台使用
2. 使用Dart语言开发
3. 底栏有`记录`和`设置`两个项目
```
├── UI底栏
│ ├── 记录
│ │ ├── 已经被记录的短险卡展示
│ │ └── 添加记录
│ └── 设置

```
4. `设置`的内容包括`导出json记录`，`导入json记录`，`颜色（跟随系统，浅色，深色）`版本号等等
5. 卡片结构
```
┌─────────────────────────┐
│  ← 新建记录    [保存]    │  ← AppBar
├─────────────────────────┤
│  姓名 *                 │
│  ┌───────────────────┐  │
│  │ 张三              │  │  ← TextField
│  └───────────────────┘  │
│  保险类型 *             │
│  ┌───────────────────┐ │
│  │ 用户填写          │  │  ← Dropdown
│  └───────────────────┘  │
│  证件号码 *             │
│  保险费用 *             │
│  保险日期 *             │
│  手机号码 *             │
│  投保人                 │
│  受保人                 │
│  保单号                 │
└─────────────────────────┘
```

## 技术栈

### 开发技术
- **框架**：Flutter
- **语言**：Dart
- **UI 规范**：Material Design 3
- **目标平台**：Android（首发）

### 项目建议技术选型
- **状态管理**：Provider（轻量，适合中小型表单与设置页场景）
- **本地存储**：Hive 或 SQLite
  - 若以快速开发为目标，优先使用 **Hive** 存储记录数据
  - 若后续需要更复杂查询、筛选、统计，可升级为 **SQLite/sqflite**
- **JSON 序列化**：json_serializable 或手写 `toJson/fromJson`
- **文件导入导出**：file_picker + path_provider
- **日期处理**：intl
- **主题管理**：Flutter ThemeMode（跟随系统 / 浅色 / 深色）
- **路由导航**：Flutter Navigator 2.0 简化方案或原生 Navigator
- **表单校验**：Flutter Form + TextFormField 校验机制

### 建议项目分层
- **presentation**：页面、组件、主题、交互逻辑
- **domain**：实体定义、业务规则
- **data**：本地存储、JSON 导入导出、数据模型映射

### 建议目录结构
```text

│   ├── routes/

```
## 阶段性任务

### 第一阶段：项目初始化与基础框架
- 创建 Flutter 项目并完成 Android 运行验证
- 配置 Material Design 3 主题
- 搭建基础目录结构
- 完成底部导航栏（记录 / 设置）
- 建立路由与页面跳转骨架

### 第二阶段：记录模块开发
- 定义短险卡数据模型
- 实现记录列表页
- 实现新建记录页与编辑记录页
- 完成表单输入、必填校验、日期选择
- 支持记录详情查看
- 支持记录编辑与删除

### 第三阶段：本地存储与搜索
- 接入本地存储方案（Hive 或 SQLite）
- 完成记录数据的新增、读取、更新、删除
- 实现首页搜索功能
- 支持按姓名、证件号、保单号等字段检索

### 第四阶段：设置与数据管理
- 实现主题切换（跟随系统 / 浅色 / 深色）
- 实现 JSON 导出功能
- 实现 JSON 导入功能
- 实现数据清空功能
- 增加关于应用与版本信息展示

### 第五阶段：体验优化与发布准备
- 优化表单交互与错误提示
- 优化列表展示与空状态页面
- 补充异常处理与导入导出边界校验
- 完成应用图标、名称、版本整理
- 进行基础测试与打包发布准备

### 里程碑验收建议
- **M1**：完成应用基础框架，可进入记录页和设置页
- **M2**：完成短险卡记录的新增、编辑、详情、删除
- **M3**：完成本地持久化与搜索功能
- **M4**：完成导入导出、主题设置、关于页面
- **M5**：完成测试优化并具备首个可交付版本



## 交互流程

```
启动应用
   ↓
首页(记录列表)
   ├─→ [+新建] → 编辑页 → 填写表单 → [保存] → 返回首页
   ├─→ [点击卡片] → 查看详情 → [编辑] → 编辑页
   └─→[搜索] → 搜索结果列表

设置页
   ├─→ 主题设置(浅色/深色/跟随系统)
   ├─→ 数据管理(备份/恢复/清空)
   ├─→ 关于应用
   ├─→ [导入] → 选择JSON文件 → 导入成功
   └─→ [导出] → 选择导出范围 → 生成JSON文件

```
